# RTX 3080 Laptop acceptance summary

This is the compact, tracked evidence for the final M5 acceptance run. It is deliberately smaller
than the raw benchmark, Compute Sanitizer, and Nsight artifacts, which remain untracked because of
their size and machine-specific formats. The adjacent JSON retains machine-readable values and
SHA-256 identities for the raw artifacts.

The stability run observed 1830.289 seconds including drain. After discarding warmup samples, the
ending GPU-memory window was 198.5 MiB lower than the starting window. Queue depth, active sequences,
and KV blocks all drained to zero. Thermal slowdown was observed, so this run is stability evidence,
not an unthrottled performance baseline.

## Fixed-workload comparison

Context 128, 32 output tokens, greedy sampling, three warmups, and ten samples:

| Engine | Total tok/s | Peak MiB | Qualification |
|---|---:|---:|---|
| PyTorch FP16 | 35.959 | 2120.872 | thermal clean |
| llama.cpp GGUF F16 | 117.734 | 3841 process-wide | thermal clean |
| TLIE FP16 | 19.019 | 2122 allocator-local | thermal clean |
| TLIE hybrid INT8 | 15.302 | 1264 allocator-local | thermal slowdown observed |

Hybrid W8A16 reduced peak device memory by `40.434%`, but throughput decreased by `20.111%`. This is
a memory result and an explicit speed non-finding, not an INT8 acceleration claim.

## Evidence boundary

- The raw artifacts are identified by SHA-256 in the adjacent JSON.
- Model weights, generated full-model traces, raw Nsight reports, and large logs are not committed.
- The tested snapshot predates the first Git commit; commit `f7c9a24` is the first committed
  implementation and contains the same executable source. Later delivery commits add CI, licensing,
  evidence, and documentation.
- Real CUDA OOM and `ResetSlot` fault injection remain unverified.
- Results cover one 16 GiB RTX 3080 Laptop GPU, one process, and bounded batching; they are not
  production-scale or multi-GPU evidence.
